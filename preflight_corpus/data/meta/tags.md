# Official Tag List

Tags provide controlled vocabulary for classifying documents. Each file should use **2–5 tags** from this list.

---

## Audits
Reports or findings from security audits.  
**Examples**:  
- `trailofbits`, `cyfrin`

## Reports
Standalone analyses or formal reports.  
**Examples**:  
- "OpenSearch Benchmarking"  
- Immunefi bounty writeups

## Standards
Official specifications or norms.  
**Examples**:  
- ERC-20, EIP-1  
- OWASP SCVS

## Guides
Instructional or explanatory documents.  
**Examples**:  
- Solidity Docs pages  
- Implementation checklists

## Checklists
Step-by-step task lists or validation items.  
**Examples**:  
- Secure coding checklist  
- Deployment readiness checklist

## Taxonomy
Classification lists or enumerations.  
**Examples**:  
- SWC registry  
- Vulnerability classes

## Code
Source code or smart contract files.  
**Examples**:  
- OpenZeppelin contracts  
- Solidity examples

---

**Note**: Tags are singular, capitalized, and stable across the repo.
